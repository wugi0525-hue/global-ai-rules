# Global Lessons Learned (전사 프로젝트 공통 레슨런)

이 문서는 앞으로 진행될 모든 신규 앱/웹 프로젝트에서 동일한 시행착오를 겪지 않기 위해 과거 프로젝트(APHASIA-APP 등)에서 겪었던 주요 이슈와 해결 방법을 누적 기록하는 문서입니다. 
새 프로젝트 기획 및 개발 시 반드시 이 문서를 먼저 참고해야 합니다.

---

## 1. Authentication (로그인 및 인증)

### 1.1 소셜 로그인 적용 시 signInWithRedirect 사용 금지 및 signInWithPopup 사용 원칙
- **문제점**: 
  - 인증 시 `signInWithRedirect` 방식을 사용하면, Localhost 개발 환경이나 사파리(Safari), 브라우저 시크릿 모드 등에서 외부 인증 화면을 거쳐 되돌아올 때 교차 출처(Cross-Origin) 보안 및 서드파티 쿠키 차단으로 인해 인증 토큰(세션)이 소실됨.
  - 이로 인해 사용자가 **정상적으로 로그인을 마쳤음에도 불구하고 세션이 없어 다시 로그인 화면으로 튕기는 무한 루프 버그**가 지속적으로 발생함 (대표님 직접 검수 중 반복 지적된 사항).
- **진짜 해결 방법 (Lesson Learned)**:
  - **위와 같은 무한 로그인 튕김 현상을 없애기 위해 향후 모든 프로젝트는 무조건 `signInWithPopup` (팝업 방식)을 사용할 것!**
  - **단, 모바일 카카오톡, 인앱 브라우저 등에서 팝업이 차단되는 문제의 진짜 해결책:** OAuth 방식을 Redirect로 바꾸는 것이 아니라, **프론트엔드 로그인 페이지(`Login.jsx`) 진입 시 `userAgent`를 감지하여 인앱 브라우저 환경일 경우 외부 브라우저(Chrome/Safari)로 앱을 강제 실행시키거나 링크를 복사해 이동하도록 유도하는 방어요소를 씌우는 것이 정답임.** (현재 Aphasia-app의 Login.jsx 우회 구조 참조).

## 2. QA and Verification Principles (검수 및 테스트 원칙)

### 2.1 AI 에이전트 다이렉트 검수 (Agentic Direct Verification) 필수화
- **발견된 문제**: 
  - 과거 로그인 로직(Redirect vs Popup)이나 대시보드 화면 배포 시, 코드를 작성한 후 실제 구동 환경(Browser)에서 클릭 테스트를 거치지 않고 "완료되었다"고 보고하여 치명적인 백색 화면(Crash)이나 무한 루프 버그가 라이브 환경에 그대로 노출됨.
  - 대표님이 이를 직접 발견하고 실망하는 상황이 반복됨.
- **해결 방안 및 절대 원칙 (Lesson Learned)**:
  - **향후 모든 UI 변경, 신규 컴포넌트 추가, 혹은 로직 수정 시 에이전트가 코드를 작성한 직후 무조건 직접 가상 브라우저(Browser Subagent)를 띄워 테스트해야 함.**
  - **테스트 기준**: 해당 페이지에 접속하여 주요 동작(클릭, 모달 팝업 오픈, 라우팅 전환, 로그인 등)을 직접 수행하여 런타임 에러(Crash)나 백색 화면 여부를 확인하고, 정상 작동을 검증한 뒤에만 대표님께 보고를 올릴 것.
  - (대표님 공식 명령에 의해 이 조항은 영구불변의 절차로 고정됨.)

### 2.2 가상 검수 임시 파일(Screenshots/Videos) 자동 폐기 원칙
- **발견된 문제**: 
  - 에이전트 다이렉트 검수(Browser Subagent)를 시행할 때마다 렌더링 확인용 스크린샷(.png)과 화면 녹화 영상(.webp) 파일들이 시스템 폴더(brain)에 계속 누적되어 로컬 하드 용량을 불필요하게 차지함.
- **해결 방안 및 절대 원칙 (Lesson Learned)**:
  - **시각적 UI 검수가 완전히 통과되고 보고를 마친 즉시, 에이전트는 백그라운드 터미널 커맨드(예: `Remove-Item`)를 실행하여 방금 생성된 테스트용 이미지/영상 찌꺼기들을 전부 영구 삭제(Garbage Collection)해야 함.**
  - 불필요한 파일이 누적되지 않도록 관리하는 것까지가 QA 검수 프로세스의 완전한 종료 조건임.

## 3. React 및 프론트엔드 환경 설정 (React & Frontend Setup)

### 3.1 Vite 기반 React 프로젝트 빌드 시 빈 화면(White Screen) 에러
- **문제점**: Vite로 만든 React 앱에서 빌드 후 배포 시 화면이 하얗게 나오는 문제 발생.
- **해결 방법 (Lesson Learned)**: `vite.config.js` 파일에서 `base: './'` 설정을 추가해주어야 빌드 후 정적 파일(Assets)의 상대 경로를 제대로 찾아 로드할 수 있음.
- **관련 프로젝트**: `rpa-app`, `stock-dashboard`

## 4. Git 및 버전 관리 (Git & Version Control)

### 4.1 GitHub 다중 프로젝트 관리 시 커밋 충돌 방지
- **문제점**: VS Code에서 여러 개의 프로젝트 폴더를 한 번에 열어두고 작업할 때, 소스 제어(Source Control) 탭에 Git 저장소 내역들이 혼재되어 의도치 않은 폴더에 커밋을 하거나 충돌이 발생할 위험이 있음.
- **해결 방법 (Lesson Learned)**: VS Code 창 하나당 **오직 한 개의 하위 프로젝트 폴더(예: `rpa-app`)만 단독으로 열어서 작업(Open Folder)**하는 것이 Repository(깃)가 꼬이는 것을 방지하는 가장 안전하고 확실한 방법임.
